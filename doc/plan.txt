Проект Firebird Streaming
===============================

Цель: разработать библиотеку для разбора логов репликации Firebird после обработки их утилитой fb_repl_print.
Написать интерфейс обработчика следующих события: старта, подтверждения и отката транзакции;
вставки, модификации и удаления записи из таблицы; установка значения генератора; выполнения DDL запроса.
Для этого интерфейса можно придумывать различные реализации, например в виде плагинов, позволяющие реагировать на эти события.

Этапы разработки:

1. Разработать базовые классы для разбора сегментов репликации, последовательного чтения сегментов из папки и пометки уже обработанных,
для исключения повторной обработки. Для обработчика сегментов репликации необходимо предусмотреть возможность фильтрации таблиц.
Проработать интерфейсы для реакции на события.

2. Разработать примеры, с различными видами реализации реакции на события разбора сегментов репликации.
Планируется реализовать следующие примеры:
- оповещения о событии или группы событий в транзакции через подсистему очередей RabbitMQ
- преобразование сегментов репликации в аналогичный вид в формате JSON (это позволит делать простой разбор сегментов сторонними инструментами)
- преобразование событий из сегментов репликации в группы DML операторов для выполнения их на какой-либо СУБД
- прямая реакция на событие с моментальным исполнением команд на одной из СУБД

3. Практическое применение потокового обработчика. Пополнение витрин и/или аналитических хранилищ вроде ClickHouse.
Сделать настраиваемую реализацию с конфигурацией.
Написание документацию для библиотеки.
